**Fetch approved records on mount:**
```typescript
useEffect(() => {
  setLoading(true);
  fetch("/api/vehicles?status=approved&page=1&pageSize=500")
    .then(res => res.json())
    .then(json => {
      if (json.success) setRecords(json.data.vehicles);
    })
    .catch(() => toast.error("Failed to load records"))
    .finally(() => setLoading(false));
}, []);
```

**handleExportNew** (D-41 mode 1, D-44, XPRT-05):
1. Set `isExporting = true`.
2. POST `/api/export` with `{ vehicleIds: Array.from(selectedIds) }`.
3. If response ok: get blob via `res.blob()`. Create download link per RESEARCH.md client-side download pattern. Extract filename from Content-Disposition header or fallback to `Garage_Register_YYYY-MM-DD.xlsx`. Trigger download. Show success state. Toast: "Garage Register exported successfully" per UI-SPEC.
4. Update records in local state: mark exported records as status="exported".
5. Clear selectedIds.
6. If response not ok: parse error, toast.error with message per UI-SPEC: "Export failed. Please try again. If the problem persists, check that all selected records have complete data."
7. Set `isExporting = false`.

**handleExportAppend** (D-41 mode 2, D-44, XPRT-05):
1. Validate appendFile exists. If not, toast.error.
2. Set `isExporting = true`.
3. Create FormData: `formData.append("file", appendFile)`. `formData.append("vehicleIds", JSON.stringify(Array.from(selectedIds)))`.
4. POST `/api/export/append` with FormData (no Content-Type header -- browser sets multipart boundary).
5. If response ok: same download trigger as above. Filename from Content-Disposition (original filename per D-44). Toast: "Records appended to {filename} successfully" per UI-SPEC.
6. Update local state same as export new.
7. If error: toast.error "Append failed. Please check the uploaded file and try again."
8. Set `isExporting = false`.

```
<Tabs value={mode} onValueChange={setMode}>

  </TabsContent>
</Tabs>
```

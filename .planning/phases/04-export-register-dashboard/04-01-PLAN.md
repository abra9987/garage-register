**`generateNewGarageRegister(records: VehicleExportData[]): Promise<Buffer>`** per D-41 mode 1:
1. Create new ExcelJS Workbook. Set `creator` to "AD Auto Garage Register".
2. Add worksheet named "Garage Register".
3. Set column widths to match Ministry template (17 columns, widths listed above).
4. Add header row at row 1: "GARAGE REGISTER / INSCRIPTION DE GARAGE" merged across C1:Q1, font Aptos Narrow 14 bold.
5. Add column headers at rows 4-5 matching Ministry template (see RESEARCH.md lines 320-324 for merged cell structure). Headers: Internal #, Purchased From (Name, Address), Vehicle Details (Make, Style, Year, Serial No., Colour), Odometer Reading, Plate No., Date into Stock, Re-sale/wrecking, Date out of Stock, Sold To (Name, Address). Apply bold font and thin borders.
6. Sort records by purchaseDate ascending. Group by month. Before each month's records, insert a month separator row: Col 3 = "YYYY MonthName" (e.g., "2026 January"), yellow fill `FFFFFF00` on the entire row, font Aptos Narrow 12.
7. For each record, create row values array (17 items) using the column mapping above. Convert purchaseDate/saleDate strings to `new Date(dateStr + "T00:00:00")` for Excel date type (per Pitfall 1). Format odometer as `odometer.toLocaleString() + " km"` string (per Pitfall 3). Year is number type. Col 14 is always "Export".
8. After `ws.addRow(rowValues)`, apply DATA_CELL_STYLE to every cell in the row. Set numFmt on date cells (cols 13 and 15).
9. Return `Buffer.from(await wb.xlsx.writeBuffer())`.

**`appendToExistingRegister(existingBuffer: Buffer, records: VehicleExportData[]): Promise<Buffer>`** per D-41 mode 2, D-42, D-43 (XPRT-02/03):
1. Create ExcelJS Workbook, load from `existingBuffer` via `wb.xlsx.load(existingBuffer)`.
2. Get the last worksheet: `wb.worksheets[wb.worksheets.length - 1]`.
3. Get last row number: `ws.lastRow?.number ?? 7`. Get style reference row: `ws.getRow(lastRowNum)`.
4. For each record, build the same 17-value row array as generateNew. Call `ws.addRow(rowValues)`.
5. Copy styles from reference row to new row: iterate `styleRefRow.eachCell({ includeEmpty: true })` and set `newRow.getCell(colNumber).style = { ...cell.style }` (per Pitfall 2).
6. Explicitly set numFmt on date cells (cols 13 and 15) to "mm-dd-yy" since style copy may not preserve numFmt.
7. Return `Buffer.from(await wb.xlsx.writeBuffer())`.

Both functions share a private helper `buildRowValues(record: VehicleExportData): (string | number | Date | null)[]` to avoid duplication.

**POST /api/export/route.ts** (XPRT-01, XPRT-04, XPRT-05, D-43, D-44):
1. Auth guard: `const session = await auth.api.getSession({ headers: request.headers })`. Return 401 if no session.
2. Parse body: `const { vehicleIds } = await request.json()`. Validate vehicleIds is a non-empty string array; return 400 if invalid.
3. Fetch records: `db.select().from(vehicles).where(and(inArray(vehicles.id, vehicleIds), eq(vehicles.status, "approved")))`. Select all fields needed by VehicleExportData: jobNumber, vin, year, make, model, color, odometer, sellerName, sellerAddress, buyerName, buyerAddress, purchaseDate, saleDate. Also select `id` for status update.
4. If no records found (all IDs invalid or not approved), return 400 "No approved records found for the given IDs".
5. Call `generateNewGarageRegister(records)` to get XLSX buffer.
6. Mark records as exported: `db.update(vehicles).set({ status: "exported", exportedAt: new Date(), updatedAt: new Date() }).where(inArray(vehicles.id, vehicleIds))`.
7. Audit log each exported record: loop through records, call `logAudit({ entityType: "vehicle", entityId: record.id, action: "exported", userId: session.user.id })`.
8. Generate filename per D-44: `Garage_Register_${format(new Date(), "yyyy-MM-dd")}.xlsx` using date-fns `format`.
9. Return `new Response(buffer, { headers: { "Content-Type": "application/vnd.openxmlformats-officedocument.spreadsheetml.sheet", "Content-Disposition": \`attachment; filename="${filename}"\` } })`.
10. Wrap in try/catch. On error, return `apiError("Export failed. Please try again.", 500)`.

**POST /api/export/append/route.ts** (XPRT-02, XPRT-03, XPRT-05, D-41 mode 2, D-44):
1. Auth guard (same pattern).
2. Parse multipart FormData: `const formData = await request.formData()`. Get `file` (the uploaded XLSX) and `vehicleIds` (JSON string array).
3. Validate: file must be present and have `.xlsx` extension or MIME type `application/vnd.openxmlformats-officedocument.spreadsheetml.sheet`. Max 10MB. vehicleIds must be non-empty string array (parse with `JSON.parse`). Return 400 on validation failure.
4. Read file to Buffer: `Buffer.from(await file.arrayBuffer())`.
5. Fetch approved records (same query as /api/export).
6. If no records found, return 400.
7. Call `appendToExistingRegister(fileBuffer, records)` to get XLSX buffer with appended rows.
8. Mark records as exported (same as /api/export).
9. Audit log each record (same as /api/export).
10. Filename per D-44: preserve original filename from upload (`file.name`).
11. Return Response with XLSX content-type and Content-Disposition using original filename.
12. Wrap in try/catch. On error, return `apiError("Append failed. Please check the uploaded file and try again.", 500)`.

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Spoofing | /api/export, /api/export/append | mitigate | Auth guard via auth.api.getSession on both routes -- reject 401 if no valid session |
| T-04-02 | Tampering | /api/export vehicleIds | mitigate | Validate vehicleIds is string array. Query only vehicles with status="approved" -- cannot export non-approved records even with valid IDs |
| T-04-03 | Tampering | /api/export/append uploaded file | mitigate | Validate file extension (.xlsx) and MIME type. Max 10MB size limit. ExcelJS.load() will throw on malformed files -- caught by try/catch |
| T-04-04 | Information Disclosure | XLSX response | accept | Export contains only vehicle data the authenticated user already has access to. Single-user app, no row-level access control needed for v1 |
| T-04-05 | Denial of Service | Large vehicleIds array | mitigate | Limit vehicleIds array to max 500 entries at API level. At 25 deals/month this is years of data |
| T-04-06 | Elevation of Privilege | Export status update | accept | Any authenticated user can export approved records. Single-user app -- no privilege escalation risk in v1 |
</threat_model>

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (gap-1) | Icon-to-text gaps inside buttons, checkbox-to-label gap |
| sm | 8px (gap-2 / p-2) | Compact element spacing, inline padding, table cell padding vertical |
| md | 16px (gap-4 / p-4) | Card padding, form field gaps, table cell padding horizontal |
| lg | 24px (gap-6 / p-6) | Page padding (matches existing main p-6), section padding |
| xl | 32px (gap-8 / p-8) | Dashboard stat card internal padding, empty state padding |
| 2xl | 48px (py-12) | Empty state vertical spacing (visual breathing room) |
| 3xl | 64px (py-16) | Not used this phase |

Exceptions: Table header row uses 12px (py-3) vertical padding for denser data display. Stat cards use 20px (p-5) as intermediate between md and lg for compact-but-readable card layout.

---

## Typography

| Role | Size | Weight | Line Height | Tailwind Class |
|------|------|--------|-------------|----------------|
| Body / Table cell | 14px | 400 (regular) | 1.5 | `text-sm font-normal leading-normal` |
| Subheading / Stat label | 16px | 600 (semibold) | 1.3 | `text-base font-semibold` |
| Page heading | 24px | 600 (semibold) | 1.2 | `text-2xl font-semibold` |
| Stat value (large) | 28px | 600 (semibold) | 1.2 | `text-3xl font-semibold` |

Exactly 2 weights used across the entire phase: **400 (regular)** for body text, labels, table data, and badge text; **600 (semibold)** for all emphasis roles (headings, subheadings, stat values, column headers).

Notes:
- Table column headers use `text-xs font-semibold uppercase tracking-wider text-muted-foreground` (12px, weight 600).
- VIN cells use `font-mono text-sm tracking-wider` for readability (consistent with Phase 2).
- Activity feed timestamps use `text-xs font-normal text-muted-foreground` (12px, weight 400).
- Stat card labels use `text-sm font-normal text-muted-foreground` (14px, weight 400).
- Stat card values use `text-3xl font-semibold` (28px, weight 600) for visual prominence.
- Pagination text uses `text-sm font-normal text-muted-foreground` (14px, weight 400).

### Search Bar (D-47)

| Property | Value |
|----------|-------|
| Component | shadcn Input with `Search` Lucide icon prefix |
| Placeholder | "Search by VIN, job number, make, model, seller, buyer..." |
| Width | `flex-1` (fills available space) |
| Debounce | 300ms before sending server query |
| Behavior | Server-side search, synced to URL via nuqs `?q=` param |
| Icon | `Search` from Lucide, `size-4 text-muted-foreground` positioned inside input left |
| Clear | `X` icon button appears when input has value, clears search |

### Filter Controls (D-48)

| Filter | Component | Options | URL Param |
|--------|-----------|---------|-----------|
| Status | shadcn Select (popover dropdown) | "All Statuses", "Pending Review", "Approved", "Exported" | `?status=` |
| Date Range | shadcn Select (popover dropdown) | "All Time", "This Month", "Last 3 Months", "Custom..." | `?dateRange=` |

Custom date range: When "Custom..." is selected, show a popover with two date inputs (from/to) using shadcn Calendar + Popover. URL params: `?dateFrom=YYYY-MM-DD&dateTo=YYYY-MM-DD`.

### Register Table (Desktop, D-46)

| Property | Value |
|----------|-------|
| Component | shadcn Table (new install needed) |
| Breakpoint | Show table at `md` (768px+), cards below |
| Row hover | `hover:bg-muted/50 transition-colors` |
| Row click | Navigates to `/vehicles/{id}/review` |
| Checkbox column | First column, for export selection |

Table columns:

| Column | Header | Width | Sortable | Content Style |
|--------|--------|-------|----------|---------------|
| Checkbox | `<Checkbox>` (header = select all) | 40px | No | shadcn Checkbox |
| Job # | "Job #" | 100px | Yes | `text-sm font-semibold` |
| VIN | "VIN" | 180px | Yes | `text-sm font-mono tracking-wider truncate` |
| Year | "Year" | 60px | Yes | `text-sm font-normal` |
| Make | "Make" | 100px | Yes | `text-sm font-normal` |
| Model | "Model" | 100px | Yes | `text-sm font-normal` |
| Status | "Status" | 120px | Yes | StatusBadge component (reuse from Phase 3) |
| Acquired | "Acquired" | 100px | Yes | `text-sm font-normal` -- format YYYY-MM-DD |
| Disposed | "Disposed" | 100px | Yes | `text-sm font-normal` -- format YYYY-MM-DD or "--" |
| Action | "" | 80px | No | Button "View" variant="ghost" size="sm" |

Sort behavior:
- Click column header to sort ascending. Click again for descending. Click third time to clear sort.
- Active sort column header shows `ArrowUp` or `ArrowDown` icon (Lucide, `size-3`) next to header text.
- Default sort: `createdAt` descending (newest first).
- Sort synced to URL via nuqs: `?sort=jobNumber&order=asc`.

Table header style:
- `text-xs font-semibold uppercase tracking-wider text-muted-foreground`
- `bg-muted/50` background
- `py-3 px-4` padding

Table cell style:
- `py-3 px-4` padding
- `border-b border-border`

## Interaction Contracts

### Dashboard Flow

1. User lands on `/dashboard` (home page after login)
2. Three stat cards load with skeleton placeholders (200ms typical)
3. Stats populate from server: Pending Review count, Exported This Month count, Total Vehicles count
4. Quick action buttons appear below stats
5. Activity feed loads below quick actions with last 20 audit log entries
6. Each activity entry shows action icon, description, and relative timestamp
7. Dashboard refreshes stats on every page load (no cache -- per specific idea in CONTEXT.md)

### Register Flow

1. User navigates to `/register`
2. Search bar and filter dropdowns render immediately
3. Table loads with skeleton rows (3 placeholder rows)
4. Server returns first page (25 records, default sort: newest first)
5. User can type in search bar -- debounced 300ms, URL updates, table refreshes
6. User can select status filter -- URL updates immediately, table refreshes
7. User can click column header to sort -- URL updates, table refreshes
8. User can check records via checkbox column (selection state is client-side, not URL-synced)
9. User clicks "View" on a row to navigate to review page
10. Pagination: "Prev"/"Next" buttons navigate pages, URL syncs `?page=N`

### Export Flow

1. User navigates to `/export`
2. Tabs render with "Export New" active by default
3. Approved records load into selection list
4. User checks individual records or "Select All"
5. Action bar shows selection count and "Export New XLSX" button enables
6. User clicks "Export New XLSX"
7. Button shows spinner + "Generating..."
8. Server generates XLSX via ExcelJS, returns as downloadable blob
9. Browser download triggers automatically
10. Success state: card background flashes green, toast appears, selected records update to "Exported" status
11. For Append mode: user first uploads existing XLSX, then selects records, then clicks "Append to Register"

### Toast Notifications

| Event | Type | Message | Duration |
|-------|------|---------|----------|
| Export success (new) | success | "Garage Register exported successfully" | 4s |
| Export success (append) | success | "Records appended to {filename} successfully" | 4s |
| Export failed | error | "Export failed. Please try again." | persistent (dismiss) |
| Invalid XLSX file | error | "Only .xlsx files are accepted" | 5s |
| XLSX too large | error | "File is too large. Maximum size is 10 MB." | 5s |

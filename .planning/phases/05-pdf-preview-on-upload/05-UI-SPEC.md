## Component Modifications

### DropZone Enhancement (existing: `src/components/upload/drop-zone.tsx`)

The DropZone's "file loaded" state is replaced with an inline PDF preview. The empty/drag-over states remain unchanged.

#### Preview Container

| Property | Value |
|----------|-------|
| Container | `div` with `relative` positioning wrapping the preview area |
| Preview area background | `bg-muted rounded-md overflow-hidden` (matches review page PdfPreview) |
| Preview width | Fill available container width |
| Preview height | Determined by PDF page aspect ratio (first page rendered at container width) |
| Min container height | `min-h-[200px]` (matches empty state) |
| Border (file loaded) | 2px solid `border-primary` (unchanged from current) |
| Internal padding | `p-4` (16px) around the preview content |
| Content alignment | `flex flex-col items-center gap-2` |

#### PDF Page Rendering

| Property | Value |
|----------|-------|
| Library | react-pdf (^10.4.1, already installed) |
| Worker | Reuse existing pdfjs worker configuration from `src/components/review/pdf-preview.tsx` |
| Page rendered | Page 1 only |
| Width | Container width minus padding (measured via `useRef` + inline calculation) |
| `renderTextLayer` | `false` (preview only -- no text selection needed) |
| `renderAnnotationLayer` | `false` (preview only -- no link clicks needed) |
| File source | `URL.createObjectURL(file)` -- client-side, no server round-trip |

Note on `renderTextLayer` and `renderAnnotationLayer`: Set to `false` for the upload preview because this is a quick visual confirmation, not a reading/interaction surface. The review page PdfPreview uses `true` for both because users need to select/copy text during review. Keeping them off reduces rendering overhead and avoids unnecessary text-layer CSS imports in the DropZone.

#### Remove Button (overlaid)

| Property | Value |
|----------|-------|
| Component | shadcn Button, variant `ghost`, size `icon` |
| Position | `absolute top-2 right-2` (8px from top-right corner of the container) |
| Icon | Lucide `X`, size-4 (16px) |
| Background | `bg-background/80 backdrop-blur-sm` for visibility over PDF content |
| Border radius | `rounded-full` for circular button |
| Dimensions | `size-8` (32px touch target) |
| `aria-label` | `"Remove file {filename}"` |
| Hover | `hover:bg-background` (fully opaque on hover) |
| Click | `e.stopPropagation()` to prevent dropzone re-open |

#### File Info (below preview)

| Property | Value |
|----------|-------|
| Filename | `text-sm font-normal` (14px, weight 400) -- truncated with `truncate` class if too long |
| File size | Inline with filename: `{filename} ({size})` in same text style |
| Layout | `text-center` below preview area |

#### Page Count Badge

| Property | Value |
|----------|-------|
| Visibility | Only shown when PDF has more than 1 page |
| Position | Below filename, inline |
| Component | shadcn Badge, variant `secondary` |
| Content | `"{N} pages"` (e.g., "3 pages") |
| Typography | `text-xs` (12px, weight 400) -- inherited from Badge defaults |

#### Loading State (Skeleton)

| Property | Value |
|----------|-------|
| Component | shadcn Skeleton |
| Dimensions | `h-[200px] w-full` (matches min-height of container) |
| Replaces | PDF render area while react-pdf loads the document |

#### Error State

| Property | Value |
|----------|-------|
| Trigger | react-pdf Document `onLoadError` callback |
| Behavior | Show toast via Sonner: "Unable to preview PDF. The file may be corrupted." (error, 5s duration). Revert drop zone to show file info without preview (filename + size + remove button, no PDF render). |
| Fallback display | Same as current Phase 2 file-loaded state: FileText icon + filename + remove button. User can still proceed to extraction. |

## Interaction Contracts

### File Selection Flow (modified from Phase 2)

1. User drags PDF onto drop zone (or clicks to browse) -- unchanged
2. File validates client-side (PDF only, max 10MB) -- unchanged
3. Valid file: drop zone transitions to preview state:
   a. Skeleton appears at `h-[200px] w-full` inside the drop zone
   b. react-pdf loads PDF from `URL.createObjectURL(file)` (client-side, no server request)
   c. On load success: first page renders at container width, skeleton disappears
   d. `numPages` callback fires -- if > 1, page count badge appears below filename
   e. Filename and size appear below preview
   f. Remove button (X) appears at top-right corner of preview area
4. User sees the actual first page of their PDF -- visual confirmation of correct document
5. If react-pdf fails to load: toast error, fallback to Phase 2 file-loaded state (icon + filename)
6. User clicks X to remove -- drop zone reverts to empty state, `URL.revokeObjectURL()` called for cleanup

### Object URL Lifecycle

| Event | Action |
|-------|--------|
| File dropped/selected | `URL.createObjectURL(file)` -- store in component state or ref |
| File removed by user | `URL.revokeObjectURL(url)` -- cleanup to prevent memory leak |
| Component unmount | `URL.revokeObjectURL(url)` in useEffect cleanup |
| New file replaces old | Revoke old URL before creating new one |

### Toast Notifications

| Event | Type | Message | Duration |
|-------|------|---------|----------|
| File uploaded | success | "AP Invoice uploaded" / "AR Invoice uploaded" | 3s (unchanged) |
| Preview load failed | error | "Unable to preview PDF. The file may be corrupted." | 5s |
| Invalid file type | error | "Only PDF files are accepted" | 5s (unchanged) |
| File too large | error | "File is too large. Maximum size is 10 MB." | 5s (unchanged) |

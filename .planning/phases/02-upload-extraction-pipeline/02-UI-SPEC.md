## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (gap-1) | Icon-to-text gaps inside buttons |
| sm | 8px (gap-2 / p-2) | Compact element spacing, inline padding |
| md | 16px (gap-4 / p-4) | Card padding, form field gaps |
| lg | 24px (gap-6 / p-6) | Page padding (matches existing main p-6), section padding |
| xl | 32px (gap-8 / p-8) | Drop zone internal padding |
| 2xl | 48px (py-12) | Drop zone vertical spacing (visual breathing room) |
| 3xl | 64px (py-16) | Not used this phase |

Exceptions: Drop zones use min-height of 200px (not a spacing token -- a component dimension for comfortable drag target area).

### Upload Drop Zones (D-14)

Two drop zones side by side in a flex row. On mobile (below 768px), stack vertically.

| Property | Value |
|----------|-------|
| Layout | `flex flex-col md:flex-row gap-4` |
| Drop zone container | Card component with dashed border |
| Min height | 200px |
| Border (idle) | 2px dashed `border-border` |
| Border (drag-over) | 2px dashed `border-primary` with `bg-primary/5` |
| Border (file-loaded) | 2px solid `border-primary` |
| Border radius | `rounded-lg` (--radius) |
| Internal padding | 32px (p-8) |
| Text alignment | Center |
| Icon | `Upload` from Lucide, size-10, `text-muted-foreground` |

Drop zone states:

1. **Empty (idle):** Dashed border, upload icon, "Drag PDF here or click to browse" text, label at top ("AP Invoice (Purchase)" / "AR Invoice (Sale)")
2. **Drag-over:** Border changes to primary color, background gets subtle primary tint (`bg-primary/5`), icon pulses
3. **File loaded:** Solid primary border, file icon replaces upload icon, filename displayed, file size displayed, "Remove" ghost button appears
4. **Invalid file:** Destructive border flash, toast notification "Only PDF files are accepted"
5. **Uploading:** Skeleton pulse overlay on the drop zone

### Extraction Status (D-20)

After clicking "Start Extraction", the action bar transforms into a status indicator.

| State | Visual |
|-------|--------|
| Idle (pre-upload) | "Start Extraction" button disabled with `opacity-50` |
| Ready (files uploaded) | "Start Extraction" button enabled, primary variant, size lg |
| Extracting | Button replaced by inline status: spinner (Lucide `Loader2` with `animate-spin`) + "Extracting data from documents..." in `text-sm text-muted-foreground`. Skeleton placeholders appear in results section. |
| Complete | Status text changes to check icon + "Extraction complete" in green for 2 seconds, then results populate below |
| Failed (D-24) | Status shows alert-triangle icon + "Extraction failed. Please try again." in destructive color. "Retry Extraction" button appears (destructive variant). |

Polling interval: 2 seconds (Claude's discretion area, per CONTEXT.md).

## Interaction Contracts

### Upload Flow

1. User lands on `/upload` page -- sees empty drop zones and disabled "Start Extraction" button
2. User drags PDF onto AP zone (or clicks to browse)
3. File validates client-side (PDF only, max 10MB) -- invalid file shows toast error
4. Valid file: drop zone transitions to "file loaded" state showing filename + size
5. Job number input is available for manual entry (or auto-populates after extraction)
6. "Start Extraction" button enables when at least one PDF is loaded
7. User clicks "Start Extraction"
8. Button transforms to extraction status with spinner
9. Skeleton placeholders appear in results section below
10. Polling every 2 seconds checks extraction status
11. On success: results populate, skeletons replaced with real data
12. On failure: error status with retry button shown

### Single vs Dual PDF (D-15, D-28)

- One PDF uploaded: Extract available fields, show results. Missing fields from other document show as "Not Found". No cross-validation section shown.
- Both PDFs uploaded: Extract from both, show all fields, show cross-validation section if conflicts exist. If no conflicts, show green checkmark "Documents are consistent".
- Second PDF added later: After initial extraction, user can upload second PDF to the empty zone. New extraction runs automatically, cross-validation appears.

### Toast Notifications

| Event | Type | Message | Duration |
|-------|------|---------|----------|
| File uploaded | success | "AP Invoice uploaded" / "AR Invoice uploaded" | 3s |
| Invalid file type | error | "Only PDF files are accepted" | 5s |
| File too large | error | "File is too large. Maximum size is 10 MB." | 5s |
| Extraction started | info | "Extracting data from documents..." | 3s |
| Extraction complete | success | "Extraction complete -- review data below" | 4s |
| Extraction failed | error | "Extraction failed. Please try again." | persistent (dismiss) |
| Upload failed | error | "Upload failed. Check your connection and try again." | persistent (dismiss) |

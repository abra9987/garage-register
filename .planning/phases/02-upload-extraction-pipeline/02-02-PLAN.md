```typescript
import type { ExtractionResult } from "@/types/extraction";

export interface FieldConflict {
  field: string;         // Human-readable field name (e.g., "VIN", "Year")
  fieldKey: string;      // API field key (e.g., "vin", "year")
  apValue: string | number | null;
  arValue: string | number | null;
}

export interface CrossValidationResult {
  hasConflicts: boolean;
  conflicts: FieldConflict[];
}
```

Export `crossValidateExtractions(apResult: ExtractionResult, arResult: ExtractionResult): CrossValidationResult`.

Fields to cross-validate (shared between AP and AR):
- `vin` (label: "VIN") -- case-insensitive comparison, trim whitespace
- `year` (label: "Year") -- numeric equality
- `make` (label: "Make") -- case-insensitive comparison, trim whitespace
- `model` (label: "Model") -- case-insensitive comparison, trim whitespace

Comparison rules:
- Skip comparison if either value is null (field not found in one document -- not a conflict per D-23)
- For strings: normalize to uppercase + trim before comparing
- For numbers: strict equality
- Only report as conflict when both values are non-null and different

Return `{ hasConflicts: false, conflicts: [] }` when no conflicts found.
Return `{ hasConflicts: true, conflicts: [...] }` with specific field details when conflicts exist.

This function is called by the API route (Plan 03) only when both AP and AR extractions exist for a vehicle (per D-28).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| PDF buffer -> Claude API | Untrusted PDF content sent to external API |
| Claude API response -> application | API response validated via Zod schema before use |
| ANTHROPIC_API_KEY | Secret stored in env var, never exposed in responses |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04 | Information Disclosure | claude-client.ts | mitigate | ANTHROPIC_API_KEY read from env var only. Never logged, never returned in API responses. Client creation throws if missing. |
| T-02-05 | Tampering | extract-document.ts | mitigate | Claude API response validated through ExtractionSchema Zod parse. Malformed/unexpected output rejected before reaching database. |
| T-02-06 | Spoofing | extraction-prompt.ts | accept | Prompt injection via PDF content is a theoretical risk. Mitigated by structured output format (constrained decoding) which limits what Claude can return to the defined schema. |
| T-02-07 | Denial of Service | extract-document.ts | mitigate | Retry limited to 1 attempt with backoff. No infinite retry loops. max_tokens set to 4096 to bound response size. |
</threat_model>

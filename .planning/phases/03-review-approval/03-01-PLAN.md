**File 1: `src/app/api/vehicles/[id]/route.ts`**

Create GET and PUT handlers.

**GET handler** -- Returns vehicle data + document metadata for the review page.
- Auth guard: `const session = await auth.api.getSession({ headers: request.headers }); if (!session) return apiError("Unauthorized", 401);`
- Async params: `const { id } = await params;`
- Query vehicle: `db.select().from(vehicles).where(eq(vehicles.id, id)).limit(1)`. Return 404 if not found.
- Query documents WITHOUT fileData (same pattern as status route -- select only id, vehicleId, type, filename, fileSize, mimeType, uploadedAt, extractionRaw): `db.select({ id: documents.id, vehicleId: documents.vehicleId, type: documents.type, filename: documents.filename, fileSize: documents.fileSize, mimeType: documents.mimeType, uploadedAt: documents.uploadedAt, extractionRaw: documents.extractionRaw }).from(documents).where(eq(documents.vehicleId, id))`
- Return: `apiSuccess({ vehicle: vehicleResult[0], documents: docs })`

**PUT handler** -- Updates vehicle fields during review (D-33 form save, D-35 audit logging).
- Auth guard + async params (same as GET).
- Parse body: `const body = await request.json()`
- Validate with vehicleSaveSchema: `const parsed = vehicleSaveSchema.parse(body)`. Wrap in try/catch -- on ZodError, return `apiError("Validation failed: " + err.issues.map(i => i.message).join(", "), 400)`.
- Fetch current vehicle to compute field diffs for audit: `const [current] = await db.select().from(vehicles).where(eq(vehicles.id, id)).limit(1)`. Return 404 if not found.
- Build update object from parsed fields, excluding undefined values (only update fields that were actually sent). Use `Object.entries(parsed).filter(([_, v]) => v !== undefined)` to build the SET clause. Always add `updatedAt: new Date()`.
- Compute audit entries (D-35): For each field in the update object, compare with `current[fieldName]`. If different, create an AuditEntry with `{ entityType: "vehicle", entityId: id, action: "updated", userId: session.user.id, fieldName, oldValue: String(current[fieldName] ?? ""), newValue: String(newValue ?? "") }`. Call `logAuditBatch(auditEntries)` if entries.length > 0.
- Execute update: `db.update(vehicles).set(updateObj).where(eq(vehicles.id, id)).returning()`.
- Return: `apiSuccess({ vehicle: updated[0] })`.

**File 2: `src/app/api/vehicles/[id]/approve/route.ts`**

Create POST (approve) and DELETE (unapprove) handlers.

**POST handler** -- Approve a vehicle record (D-36, D-37).
- Auth guard + async params.
- Fetch current vehicle. Return 404 if not found.
- Check status is "pending_review": `if (current.status !== "pending_review") return apiError("Vehicle must be in pending_review status to approve", 400)`.
- Validate required fields using vehicleApprovalSchema on the current vehicle data. Extract relevant fields from `current` and pass to schema parse. On validation failure, return 400 with specific missing field message.
- Update: `db.update(vehicles).set({ status: "approved", updatedAt: new Date() }).where(eq(vehicles.id, id))`.
- Audit log: `logAudit({ entityType: "vehicle", entityId: id, action: "updated", userId: session.user.id, fieldName: "status", oldValue: "pending_review", newValue: "approved" })`.
- Return: `apiSuccess({ status: "approved" })`.

**DELETE handler** -- Unapprove a vehicle record (D-38).
- Auth guard + async params.
- Fetch current vehicle. Return 404 if not found.
- Check status is "approved" (NOT "exported"): `if (current.status !== "approved") return apiError("Only approved records can be returned to review", 400)`.
- Update: `db.update(vehicles).set({ status: "pending_review", updatedAt: new Date() }).where(eq(vehicles.id, id))`.
- Audit log: `logAudit({ entityType: "vehicle", entityId: id, action: "updated", userId: session.user.id, fieldName: "status", oldValue: "approved", newValue: "pending_review" })`.
- Return: `apiSuccess({ status: "pending_review" })`.

**File 3: `src/app/api/vehicles/[id]/documents/[docId]/content/route.ts`**

Create GET handler to serve raw PDF binary for react-pdf preview (Pattern 6 from RESEARCH).

- Query document: `db.select({ fileData: documents.fileData, filename: documents.filename, mimeType: documents.mimeType }).from(documents).where(and(eq(documents.id, docId), eq(documents.vehicleId, id))).limit(1)`.

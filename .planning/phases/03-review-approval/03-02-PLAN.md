**FILE 5: `src/app/api/vehicles/[id]/documents/[docId]/reextract/route.ts`** -- Re-extraction API endpoint (D-32).

This endpoint re-processes an existing document through the Claude API extraction pipeline without requiring a new upload.

- Auth guard: `const session = await auth.api.getSession({ headers: request.headers }); if (!session) return apiError("Unauthorized", 401);`
- Async params: `const { id, docId } = await params;` with type `{ params: Promise<{ id: string; docId: string }> }`
- Read document WITH fileData: `db.select().from(documents).where(and(eq(documents.id, docId), eq(documents.vehicleId, id))).limit(1)` -- the AND condition prevents IDOR.
- Return 404 if not found.
- Set vehicle status to "extracting": `db.update(vehicles).set({ status: "extracting", updatedAt: new Date() }).where(eq(vehicles.id, id))`
